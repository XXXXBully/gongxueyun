# Roadmap

本文档记录后续计划。优先级会根据实际使用反馈、线上风险和维护成本调整。

## 近期计划

- **补卡体验完善：** 展示每次补卡请求的明细结果，包括日期、类型、状态、失败原因和代理切换次数。
- **部署体验完善：** 提供包含 MySQL 的完整 Docker Compose 示例，降低首次部署门槛。
- **配置导入导出：** 支持导出用户配置模板，并批量导入用户和常用配置。
- **执行日志增强：** 支持按任务类型、执行状态、租户和用户备注过滤。
- **文档截图维护：** 每个核心页面保留最新截图，并在版本发布时同步更新。

## 中期计划

- **通知渠道扩展：** 在 SMTP 之外支持更多推送渠道。
- **报告模板管理：** 支持自定义日报、周报、月报 Prompt 和模板。
- **批量任务看板：** 提供更细粒度的批量任务进度、失败重试和导出能力。
- **权限细化：** 将管理端权限拆得更细，便于多人协作和租户差异化授权。
- **测试体系扩展：** 增加前端组件测试、接口级集成测试和关键业务回归用例。

## 长期计划

- **多租户隔离增强：** 让团队、租户、管理员和用户边界更清晰。
- **多平台适配：** 在现有工学云能力之外，抽象更多实习平台接入层。
- **可观测性增强：** 扩展指标、告警、结构化日志和排障手册。
- **镜像发布稳定化：** 提供更清晰的版本镜像、升级说明和回滚路径。

## 不在近期范围

- 不内置公开在线 Demo 账号。
- 不提供对第三方平台规则的规避说明。
- 不把前端组件测试、端到端测试和完整可观测性一次性全部补齐。

## 反馈方式

- Bug：使用 GitHub Issue 的 Bug 模板。
- 功能建议：使用 Feature Request 模板。
- PR：请先阅读 [CONTRIBUTING.md](./CONTRIBUTING.md)。
