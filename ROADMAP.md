# Roadmap

本文档记录后续计划。优先级会根据实际使用反馈、线上风险和维护成本调整。

## 近期计划

| 优先级 | 方向 | 目标 | 验收标准 |
|--------|------|------|----------|
| P0 | 补卡体验完善 | 展示每次补卡请求的明细结果 | 日期、类型、状态、失败原因、代理切换次数都能在界面查看 |
| P0 | 部署体验完善 | 提供包含 MySQL 的完整 Docker Compose 示例 | 新用户按文档可从空机器启动完整服务 |
| P1 | 配置导入导出 | 导出用户配置模板，批量导入用户和常用配置 | 导入失败有行级错误，成功导入可回滚 |
| P1 | 执行日志增强 | 支持按任务类型、执行状态、账号备注过滤 | 管理员能快速定位失败任务 |
| P1 | 文档截图维护 | 核心页面截图保持最新 | 发布前截图和当前 UI 一致 |

## 中期计划

| 优先级 | 方向 | 目标 | 验收标准 |
|--------|------|------|----------|
| P1 | 通知渠道扩展 | 在 SMTP 之外支持更多推送渠道 | 全局配置和个人推送配置边界清晰 |
| P1 | 报告模板管理 | 支持自定义日报、周报、月报 Prompt 和模板 | 模板可预览、可回滚、可按类型启用 |
| P2 | 批量任务看板 | 提供更细粒度的进度、失败重试和导出能力 | 批量任务状态不需要查数据库也能看懂 |
| P2 | 权限细化 | 管理端权限拆分到更细动作 | 不同管理员只能访问授权功能 |
| P2 | 测试体系扩展 | 增加前端组件测试、接口级集成测试和关键业务回归 | 核心页面和关键 API 有自动化覆盖 |

## 长期计划

| 方向 | 目标 | 备注 |
|------|------|------|
| 多平台适配 | 在现有工学云能力之外，抽象更多实习平台接入层 | 先抽接口边界，再考虑新平台 |
| 可观测性增强 | 扩展指标、告警、结构化日志和排障手册 | 重点减少线上问题定位时间 |
| 数据生命周期 | 提供日志、指标、备份的保留策略和清理工具 | 避免长期运行后数据库膨胀 |

## 不在近期范围

| 内容 | 原因 |
|------|------|
| 公开在线 Demo 账号 | 账号、验证码、外部平台调用和滥用风险不可控 |
| 第三方平台规则规避说明 | 项目只记录当前功能实现，不提供规避说明 |
| 一次性补齐完整 E2E 体系 | 成本高，先覆盖核心 API 和关键前端静态质量 |
